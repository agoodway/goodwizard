# Plan: Rename One-Shot Scheduled Tasks to One-Time Tasks

## Intent
- Perform a hard rename from one-shot terminology to one-time task terminology across runtime code, tests, docs, and OpenSpec content.
- Do not keep backward-compat aliases.
- Keep existing scheduled-task signal contract unchanged (`jido.scheduled_task_tick`, `scheduled_task.task`).

## Scope Decision (confirmed)
- Hard rename everything, including:
  - tool/action names
  - module and file names
  - storage path and persisted ID prefix
  - tests and OpenSpec text

## Naming Map (authoritative)
- `Goodwizard.Actions.Scheduling.OneShot` -> `Goodwizard.Actions.Scheduling.OneTime`
- `schedule_oneshot_task` -> `schedule_one_time_task`
- `Goodwizard.Actions.Scheduling.CancelOneShot` -> `Goodwizard.Actions.Scheduling.CancelOneTime`
- `cancel_oneshot_job` -> `cancel_one_time_job`
- `Goodwizard.Actions.Scheduling.ListOneShotJobs` -> `Goodwizard.Actions.Scheduling.ListOneTimeJobs`
- `list_oneshot_jobs` -> `list_one_time_jobs`
- `Goodwizard.Scheduling.OneShotStore` -> `Goodwizard.Scheduling.OneTimeStore`
- `Goodwizard.Scheduling.OneShotLoader` -> `Goodwizard.Scheduling.OneTimeLoader`
- `Goodwizard.Scheduling.OneShotRegistry` -> `Goodwizard.Scheduling.OneTimeRegistry`
- `lib/goodwizard/actions/scheduling/one_shot.ex` -> `lib/goodwizard/actions/scheduling/one_time.ex`
- `lib/goodwizard/actions/scheduling/cancel_oneshot.ex` -> `lib/goodwizard/actions/scheduling/cancel_one_time.ex`
- `lib/goodwizard/actions/scheduling/list_oneshot_jobs.ex` -> `lib/goodwizard/actions/scheduling/list_one_time_jobs.ex`

- `lib/goodwizard/scheduling/oneshot_registry.ex` -> `lib/goodwizard/scheduling/one_time_registry.ex`
- startup helper `reload_oneshot_jobs/0` -> `reload_one_time_jobs/0`
- storage path `workspace/scheduling/oneshot/` -> `workspace/scheduling/one_time/`
- persisted ID prefix `oneshot_` -> `one_time_`

## Execution Plan

### 1) Rename files/modules/actions first
- Use `git mv` for all runtime one-shot files to one-time file names.
- Rename matching test files under:
  - `test/goodwizard/actions/scheduling/*oneshot*`
  - `test/goodwizard/scheduling/*oneshot*`
- Update `defmodule` names, aliases, and call sites to new modules.
- Update tool names and descriptions in action modules.

### 2) Update app and agent wiring
- Update `lib/goodwizard/agent.ex` tools list to new OneTime action modules.
- Update `lib/goodwizard/application.ex` aliases and startup reload function (`reload_one_time_jobs/0`).
- Update log/error text to say "one-time task" instead of "one-shot".

### 3) Update storage path and persisted IDs
- Change store directory from `scheduling/oneshot` to `scheduling/one_time`.
- Change job ID generation/validation from `oneshot_<hex>` to `one_time_<hex>`.
- Update all loader/store/registry assumptions and tests to match new prefix/path.

### 4) Add startup migration for persisted legacy one-shot data
- Add migration before one-time reload runs:
  - if old path exists and new path does not, migrate old dir to new dir.
  - if both paths exist, apply deterministic handling (prefer fail-fast or explicit warning + skip overwrite).
  - if old path does not exist, no-op.
- During migration, update persisted IDs/files if needed to new `one_time_` prefix.
- Ensure runtime reads/writes only new path after migration.

### 5) Update tests and docs/specs
- Update all one-shot action/scheduling tests for new names and expectations.
- Update user-facing docs (`README.md`, setup/bootstrap copy) to "one-time task" wording.
- Update OpenSpec changes that mention one-shot/oneshot terminology to one-time task language and renamed action names.

### 6) Repository cleanup and validation loop
- Run broad searches for leftover `one-shot|oneshot|one_shot|OneShot` tokens.
- Resolve compile/test failures and iterate until green.

## Risks and Mitigations
- Missed string references -> run exhaustive token searches before/after and fix in one pass.
- Migration conflict/data loss risk -> deterministic conflict policy + no silent merges + test coverage.
- Persisted ID mismatch after rename -> migrate IDs and validate loader/store regex logic with tests.
- OpenSpec/docs drift -> include OpenSpec updates in same change and run final search across `openspec/**`.
